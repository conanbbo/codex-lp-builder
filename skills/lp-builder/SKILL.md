# lp-builder — 한국형 모바일 LP 제품판 빌드 스킬

이 스킬은 승인된 `lp-spec.md`를 기준으로 모바일 LP를 실제 파일로 만드는 역할을 합니다. 목표는 단순 시안이 아니라 한국 실무자가 판매·교육·배포에 바로 활용할 수 있는 제품판 LP입니다.

## 전제

- `lp-spec.md` 없이 시작하지 않습니다.
- CTA, 오퍼, 타깃, 법적 제약은 `lp-spec.md`를 기준으로 합니다.
- 이미지 생성, HTML 제작, QA, 배포 준비 순서로 진행합니다.
- `reference/design-system.md`, `reference/cta-patterns.md`, `reference/compliance-templates.md`, `reference/deployment-checklist.md`를 함께 참고합니다.

## Phase 1 — 비주얼 슬라이스 설계

모바일 LP를 4~8개의 세로형 슬라이스로 나눕니다.

추천 구조:
1. 히어로/FV
2. 문제 공감
3. 해결책/상품 소개
4. 핵심 혜택
5. 신뢰 근거/후기
6. 구성/가격/보너스
7. FAQ
8. 최종 CTA

이미지 프롬프트는 한국어 또는 영어로 작성하되, 한국어 문구가 이미지 안에 많이 들어가면 오타 가능성이 있으므로 긴 문장은 HTML 텍스트로 처리합니다.

## Phase 2 — GPT Image 2.0 이미지 생성

- 세로형 비율을 우선합니다.
- 권장 크기: `1024x1536` 또는 긴 모바일형 비율
- 이미지 안에는 짧은 제목, 분위기, 제품/인물/배경을 넣습니다.
- 가격, 날짜, 환불 조건, 긴 설명은 HTML 텍스트로 둡니다.
- 의료·미용·금융 등 민감 업종 이미지는 과장되거나 오해를 줄 수 있는 전후 비교를 피합니다.

## Phase 3 — HTML 기능화

`index.html`을 생성합니다.

필수 조건:
- 전체 폭: `max-width: 430px; width: 100%; margin: 0 auto;`
- 하단 고정 CTA
- CTA는 실제 `<a>` 또는 `<button>`
- 이미지에는 `alt` 제공
- 주요 텍스트는 이미지에만 의존하지 않기
- 미확정 링크는 임시 모달 또는 `#` 처리 후 확인 필요 표시
- 폼이 있으면 개인정보 수집 동의 문구, 수집 항목, 이용 목적, 보유 기간, 처리방침 URL 확인 항목 포함

## Phase 4 — 제품판 QA

`references/qa-checklist.md`와 아래 기준을 함께 적용합니다.

- 기능 QA: CTA, 폼, 링크, 모달, 전화/카카오톡 연결
- 디자인 QA: 390px/430px 모바일 폭, 가독성, 여백, 대비
- 카피 QA: 타깃 공감, 행동 유도, 중복 표현, 한국어 자연스러움
- 법적 QA: 과장 표현, 후기 근거, 개인정보, 가격/환불/기간 조건
- 배포 QA: 상대 경로, 미확정 링크, 공개 전 승인 문구

점검 후 수정 → 재점검을 반복합니다.

## Phase 5 — 배포

배포 전 질문:

```text
배포 방식은 어떤 것으로 진행할까요?
1) GitHub Pages
2) Vercel
```

사용자가 선택하기 전에는 공개하지 않습니다.

배포 직전에는 반드시 아래 문구로 승인받습니다.

```text
배포 전 최종 확인입니다.
공개 후 누구나 접속할 수 있습니다.
CTA 링크, 개인정보 문구, 가격/환불/후기 근거 확인 항목을 점검했습니다.
이 상태로 배포할까요? 진행하려면 “배포 승인”이라고 답해 주세요.
```

## 금지 사항

- `lp-spec.md`에 없는 보장 문구 추가 금지
- 실제 후기처럼 보이는 가짜 후기 작성 금지
- 근거 없는 수치 삽입 금지
- 개인정보 수집 폼을 만들면서 처리방침 안내를 누락하지 않기
- 사용자 승인 없이 배포하지 않기
- 미확정 링크가 남아 있는데 완성 또는 배포라고 말하지 않기
