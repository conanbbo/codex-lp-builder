# 한국형 모바일 LP 제품판 QA 체크리스트

이 체크리스트는 “초안 확인”이 아니라 판매·교육·배포 가능한 제품판 기준입니다. 각 항목은 0~100점으로 평가하고, 100점 미만이면 수정 후 재점검합니다.

## 1. 기능

- CTA 버튼이 정상 작동하는가?
- 모든 링크가 실제 목적지로 연결되는가?
- 폼 제출 또는 문의 동선이 명확한가?
- 전화, 이메일, 카카오톡 링크가 올바른가?
- 미확정 링크가 있으면 사용자 확인 항목으로 표시했는가?
- 모달, 앵커 이동, 하단 고정 CTA가 충돌하지 않는가?

## 2. 모바일 UI

- 360px~430px 폭에서 깨지지 않는가?
- 버튼 터치 영역이 최소 48px 이상인가?
- 하단 고정 CTA가 본문을 과하게 가리지 않는가?
- 폰트 크기가 너무 작지 않은가?
- 섹션 간 여백이 자연스러운가?
- 가로 스크롤이 발생하지 않는가?

## 3. 전환 구조

- 첫 화면에서 무엇을 제공하는지 3초 안에 이해되는가?
- CTA가 한 가지 행동으로 집중되는가?
- 문제 공감 → 해결책 → 근거 → 혜택 → CTA 흐름이 자연스러운가?
- 가격/혜택/신청 조건이 명확한가?
- FAQ가 주요 불안을 해소하는가?
- 타깃이 구체적으로 느껴지는가?

## 4. 카피

- 한국어 문장이 자연스러운가?
- 타깃 고객이 구체적으로 느껴지는가?
- 혜택이 기능 나열에 그치지 않는가?
- 제목과 버튼 문구가 행동을 유도하는가?
- 같은 표현이 과하게 반복되지 않는가?
- 근거가 없는 수치나 결과 단정이 없는가?

## 5. 법적 리스크

- 근거 없는 1위, 최고, 보장 표현이 없는가?
- 건강, 미용, 금융, 교육 분야의 금지 표현을 피했는가?
- 후기는 실제 근거가 있는 경우에만 사용했는가?
- 개인정보 수집 시 처리방침 안내가 있는가?
- 가격, 환불, 기간, 조건이 오해 없이 표시되는가?
- 전문가 검토가 필요한 항목을 숨기지 않고 표시했는가?

## 6. 속도와 접근성

- 이미지 용량이 과도하지 않은가?
- 이미지에 alt 텍스트가 있는가?
- 핵심 정보가 이미지에만 박혀 있지 않은가?
- 대비가 낮아 읽기 어려운 영역이 없는가?
- 불필요한 외부 스크립트를 줄였는가?
- 키보드/스크린리더 사용자가 CTA 의미를 이해할 수 있는가?

## 7. 배포 준비

- `index.html`이 루트 또는 배포 지정 폴더에 있는가?
- 상대 경로가 깨지지 않는가?
- 미확정 링크가 남아 있지 않은가?
- 공개 전 사용자 승인 문구를 받았는가?
- GitHub Pages 또는 Vercel 배포 방식이 정해졌는가?
- 배포 후 접속 URL을 확인할 계획이 있는가?

## 판정

| 점수 | 판정 |
|---:|---|
| 100 | 제품판 통과 |
| 90~99 | 실사용 가능하지만 배포 전 보완 필요 |
| 80~89 | 초안 통과, 납품/교육 전 수정 필요 |
| 79 이하 | 재작업 필요 |

완성 보고에는 반드시 기능, 디자인, 모바일·속도, 카피·법적 리스크, 배포 준비 점수를 모두 표시합니다.
