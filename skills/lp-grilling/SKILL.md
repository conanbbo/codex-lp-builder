# lp-grilling — 한국형 LP 제품판 사전 인터뷰 스킬

이 스킬은 LP를 바로 만들지 않고, 사용자의 사업과 제안을 정확히 파악해 **`lp-spec.md`**를 완성하는 역할을 합니다. 목표는 예쁜 페이지가 아니라 한국 실무자가 판매·교육·배포에 바로 사용할 수 있는 전환형 모바일 LP 사양서입니다.

## 원칙

- 질문은 반드시 **한 번에 하나**만 합니다.
- 각 질문에는 사용자가 고르기 쉬운 **추천안**을 함께 제공합니다.
- 사용자가 모호하게 답하면 임의로 확정하지 말고 “확인 필요”로 표시합니다.
- 참고 URL, 이미지, 기존 문구, 상품 설명서가 있으면 먼저 읽고 질문합니다.
- 최종 목적은 예쁜 페이지가 아니라 **전환되는 한국어 모바일 LP**입니다.
- 개인정보 수집, 후기, 성과 수치, 가격/환불/기간 조건은 반드시 확인 항목으로 분리합니다.

## 질문 순서

1. 사업/상품 내용
2. LP의 목적
3. 타깃 고객 1명
4. 오퍼와 가격
5. CTA 1개
6. 고객의 문제와 욕구
7. 차별점
8. 신뢰 근거
9. 참고 URL/디자인 톤
10. 유입 경로
11. 개인정보 수집 여부
12. 법적·운영 제약
13. 배포 방식 후보
14. 최종 승인

## 질문 방식

항상 아래 형식을 사용합니다.

```text
질문 N. [질문]

추천안:
1) ...
2) ...
3) ...

편하게 번호로 답해도 됩니다.
```

## 필수 확인 질문

폼이나 상담 신청을 받을 경우 반드시 묻습니다.

```text
신청/문의 과정에서 어떤 개인정보를 받을 예정인가요?

추천안:
1) 이름, 연락처, 문의 내용
2) 회사명, 이름, 이메일, 문의 내용
3) 카카오톡으로만 상담하고 별도 폼은 없음
4) 아직 미정
```

후기나 성과 수치를 사용할 경우 반드시 묻습니다.

```text
후기나 성과 수치는 실제 근거가 있나요?

추천안:
1) 실제 후기와 사용 허가가 있음
2) 내부 통계가 있으나 기간/표본 확인 필요
3) 아직 없음. 후기처럼 보이는 문구는 사용하지 않기
```

## 완료 조건

- CTA가 1개로 정리되어야 합니다.
- 타깃이 “모든 사람”이 아니라 구체적인 1명으로 좁혀져야 합니다.
- 근거 없는 과장 문구가 제거되어야 합니다.
- 개인정보 수집 여부와 처리방침 확인 항목이 정리되어야 합니다.
- 후기/성과 수치의 실제 근거 여부가 표시되어야 합니다.
- 사용자가 `lp-spec.md`를 확인하고 승인해야 다음 단계로 넘어갑니다.
