# /goal — 한국형 모바일 LP 제품판 품질 루프

이 문서는 `lp-spec.md`가 확정된 뒤 사용하는 품질 루프 지시문입니다. 목표는 “그럴듯한 초안”이 아니라 한국 실무자가 바로 판매·교육·배포에 활용할 수 있는 제품판 LP입니다.

## 사용 조건

- `lp-spec.md`가 있어야 합니다.
- 사용자가 승인한 CTA, 타깃, 오퍼, 톤앤매너를 변경하지 않습니다.
- 루프 중 임의로 새 기능을 추가하지 않습니다.
- 개인정보 수집, 결제, 외부 폼, 배포 작업은 사용자 승인 전 실행하지 않습니다.

## /goal 계약 템플릿

```text
/goal
lp-spec.md를 기준으로 한국어 모바일 랜딩페이지를 완성하세요.

목표:
- 스마트폰 기준으로 전환에 최적화된 LP 제작
- GPT Image 2.0 기반 비주얼 슬라이스 생성
- HTML/CSS/JS로 CTA와 폼/링크 기능화
- 기능, 디자인, 모바일·속도, 카피·법적 리스크, 배포 준비 관점 QA
- 모든 관점이 100점에 도달할 때까지 수정

필수 조건:
- lp-spec.md 범위를 벗어나지 말 것
- CTA는 1개를 중심으로 유지할 것
- 한국어 카피를 자연스럽게 작성할 것
- reference/cta-patterns.md의 업종별 CTA 기준을 참고할 것
- reference/design-system.md의 모바일 기준을 적용할 것
- reference/compliance-templates.md의 금지·주의 표현을 점검할 것
- reference/deployment-checklist.md를 배포 전 확인할 것
- 근거 없는 과장 표현을 쓰지 말 것
- 배포 전 사용자 승인을 받을 것
```

## 서브 에이전트 역할

### 1. 기능 QA

검토 항목:
- CTA 버튼 작동
- 링크 작동
- 폼 또는 문의 동선
- 영상/이미지 로딩
- 오류 없는 HTML 구조
- 미확정 링크의 확인 필요 표시

### 2. 디자인 QA

검토 항목:
- 첫 화면 임팩트
- 모바일 가독성
- 색상과 여백
- 섹션 흐름
- 브랜드 톤 일관성
- 390px/430px 폭 대응

### 3. 모바일·속도 QA

검토 항목:
- 360px~430px 폭에서 자연스러운 표시
- 이미지 용량
- 고정 CTA 위치
- 터치 영역 크기
- 불필요한 스크립트 제거
- 접근성 대체 텍스트

### 4. 카피·법적 리스크 QA

검토 항목:
- 한국어 문장 자연스러움
- 타깃 문제 공감
- CTA 설득력
- 표시광고법상 과장 표현
- 개인정보/결제/후기 관련 고지
- 의료·미용·금융·교육 업종별 주의 표현

### 5. 배포 준비 QA

검토 항목:
- `index.html` 단독 실행 가능 여부
- 상대 경로 오류 여부
- GitHub Pages/Vercel 배포 가능 구조
- 공개 전 미확정 링크 목록
- 개인정보 처리방침 URL 확인 필요 여부
- 최종 승인 전 외부 전송 없음

## 루프 규칙

1. 각 에이전트는 0~100점으로 평가합니다.
2. 100점 미만이면 구체적인 수정 지시를 냅니다.
3. 수정 후 같은 관점으로 다시 평가합니다.
4. 모든 관점이 100점이 되기 전에는 완료라고 말하지 않습니다.
5. 3회 이상 같은 문제가 반복되면 사용자 확인이 필요한 쟁점으로 분리합니다.
6. 법적 판단이 필요한 항목은 “전문가 검토 필요”로 표시하고 임의 확정하지 않습니다.
