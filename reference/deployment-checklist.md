# 한국형 LP 배포 전 체크리스트

공개 배포 전에는 반드시 이 문서를 기준으로 확인합니다.

## 1. 파일 구조

- `index.html`이 존재하는가?
- 필요한 이미지와 CSS/JS 파일이 상대 경로로 연결되어 있는가?
- 로컬에서 브라우저로 열었을 때 깨지지 않는가?
- 불필요한 테스트 파일이나 내부 메모가 포함되지 않았는가?

## 2. CTA 연결

- CTA 링크가 실제 목적지로 연결되는가?
- 카카오톡/전화/이메일/폼 링크가 정확한가?
- 미확정 링크가 `#`로 남아 있으면 배포하지 않았는가?
- 외부 폼으로 이동할 경우 개인정보 안내가 충분한가?

## 3. 모바일 확인

- 360px, 390px, 414px, 430px에서 깨지지 않는가?
- 하단 고정 CTA가 본문을 가리지 않는가?
- 버튼 터치 영역이 충분한가?
- 이미지가 너무 늦게 뜨지 않는가?

## 4. 법적/운영 확인

- 개인정보 수집 항목, 목적, 보유 기간이 표시되어 있는가?
- 개인정보 처리방침 URL이 있는가?
- 가격, 기간, 환불, 제공 조건이 오해 없이 표시되어 있는가?
- 후기와 성과 수치의 실제 근거가 있는가?
- 의료, 미용, 금융, 투자, 교육 업종의 제한 표현이 없는가?

## 5. 배포 방식별 확인

### GitHub Pages

- 저장소 루트 또는 `docs/`에 `index.html`이 있는가?
- Settings → Pages 설정이 완료되었는가?
- 공개 URL 접속 확인을 했는가?

### Vercel

- 프로젝트 루트가 올바른가?
- 빌드 명령이 필요 없는 정적 사이트인지 확인했는가?
- 배포 URL 접속 확인을 했는가?

## 6. 최종 승인 문구

배포 전 사용자에게 아래처럼 확인합니다.

```text
배포 전 최종 확인입니다.
1. CTA 링크가 실제로 연결되어 있습니다.
2. 개인정보/가격/환불/후기 문구 확인 항목을 표시했습니다.
3. 공개 후 누구나 접속할 수 있습니다.

이 상태로 배포할까요? 진행하려면 “배포 승인”이라고 답해 주세요.
```

## 7. 실패 기준

아래 중 하나라도 있으면 배포하지 않습니다.

- CTA 링크 미확정
- 개인정보 처리방침 URL 미확정인데 폼을 직접 수집함
- 근거 없는 수치·후기·보장 문구 포함
- 모바일에서 레이아웃 깨짐
- 사용자 배포 승인 없음
