# 한국형 LP 실행 테스트 시나리오

이 문서는 교육, 검수, 배포 전 데모에 사용할 수 있는 테스트 시나리오입니다.

## 테스트 1. AI 강의 신청 LP

입력 예시:

```text
상품은 직장인을 위한 ChatGPT 업무 자동화 2시간 실습 강의입니다.
CTA는 무료 상담 신청하기입니다.
타깃은 업무 시간이 부족한 30대 사무직입니다.
```

기대 산출물:

- `lp-spec.md` 생성
- 강의 신청형 섹션 구성
- 무료 상담 신청 CTA 통일
- 교육 성과 보장 표현 제거
- 개인정보 수집 문구 확인

통과 기준:

- “업무 시간을 줄일 수 있다”는 기대 표현은 가능하지만 “무조건 2배 빨라짐” 같은 표현은 없음
- 신청 폼에 개인정보 동의 문구가 있음

## 테스트 2. 로컬 서비스 예약 LP

입력 예시:

```text
상품은 동네 스튜디오 프로필 촬영 패키지입니다.
CTA는 카카오톡 예약 문의입니다.
타깃은 이직 준비용 프로필 사진이 필요한 직장인입니다.
```

기대 산출물:

- 카카오톡 CTA
- 촬영 구성/가격/예약 가능 시간 표시
- 위치와 상담 시간 확인 항목
- 전후 비교나 보정 표현 과장 없음

## 테스트 3. B2B SaaS 문의 LP

입력 예시:

```text
상품은 중소기업용 내부 문서 검색 AI입니다.
CTA는 데모 신청하기입니다.
타깃은 사내 자료 검색 시간이 오래 걸리는 50인 규모 회사의 팀장입니다.
```

기대 산출물:

- 데모 신청 CTA
- 보안/도입/연동 FAQ
- 과장된 생산성 수치 대신 확인 필요 표시
- B2B 신뢰 근거 섹션 포함

## 테스트 4. 전자책 다운로드 LP

입력 예시:

```text
상품은 초보자를 위한 노션 업무 템플릿 전자책입니다.
CTA는 샘플 PDF 다운로드입니다.
```

기대 산출물:

- 자료 다운로드 CTA
- 이메일 수집 시 개인정보 문구
- 미리보기/목차/대상자 명확화

## 테스트 5. 병원/클리닉 상담 예약 LP

입력 예시:

```text
상품은 피부 상담 예약 페이지입니다.
CTA는 상담 예약하기입니다.
```

기대 산출물:

- 의료적 효능 단정 금지
- 개인차 고지
- 후기/전후 사진 사용 시 확인 필요
- 전문가 검토 필요 표시

## 테스트 판정표

| 항목 | 통과 기준 |
|---|---|
| 인터뷰 | 질문을 한 번에 하나씩 진행 |
| 사양서 | `lp-spec.md`에 CTA, 타깃, 오퍼, 법적 주의 포함 |
| 디자인 | 390px/430px 모바일 기준 충족 |
| CTA | 하나의 행동으로 통일 |
| 법적 안전 | 근거 없는 보장·후기·성과 표현 없음 |
| 배포 준비 | 배포 전 사용자 승인 요청 |
