# 한국형 모바일 LP 디자인 시스템

이 문서는 한국 실무자가 바로 납품·교육·배포에 쓸 수 있는 모바일 랜딩페이지 기본 디자인 기준입니다.

## 1. 화면 폭

- 기본 컨테이너: `width: 100%; max-width: 430px; margin: 0 auto;`
- 검수 폭: 360px, 390px, 414px, 430px
- PC 화면에서는 중앙 정렬하고, 모바일에서는 전체 화면처럼 보이게 합니다.
- 가로 스크롤이 발생하면 실패입니다.

## 2. 타이포그래피

| 요소 | 권장 크기 | 기준 |
|---|---:|---|
| 히어로 제목 | 28~34px | 첫 화면에서 즉시 읽혀야 함 |
| 섹션 제목 | 24~30px | 한 섹션의 핵심 메시지 |
| 본문 | 15~17px | 모바일에서 읽기 쉬운 크기 |
| 보조 설명 | 13~15px | 부가 정보, 고지 문구 |
| 버튼 | 16~18px | 터치와 가독성 모두 고려 |

- 줄간격은 본문 기준 1.55~1.75를 권장합니다.
- 긴 문장은 2~3줄 단락으로 나눕니다.
- 이미지 안에 긴 한국어 문장을 넣지 않습니다.

## 3. 여백

- 섹션 상하 여백: 48~72px
- 카드 내부 여백: 18~24px
- 버튼 상하 패딩: 14~18px
- 제목과 본문 간격: 12~18px
- 섹션 사이 CTA 반복 간격: 2~4섹션마다 1회

## 4. CTA 버튼

- 최소 높이: 48px
- 권장 높이: 52~60px
- 모서리: 12~999px 중 브랜드 톤에 맞게 선택
- 하단 고정 CTA를 사용할 경우 본문 하단에 충분한 패딩을 둡니다.
- CTA는 기본적으로 1개 행동으로 통일합니다.

## 5. 섹션 구조

기본 8섹션:

1. 히어로/FV: 대상, 약속, CTA
2. 문제 공감: 고객의 현재 상황
3. 해결책: 서비스/상품 소개
4. 핵심 혜택: 결과 중심 3~5개
5. 신뢰 근거: 후기, 사례, 숫자, 경력
6. 구성/가격: 제공 내용과 조건
7. FAQ: 망설임 해소
8. 최종 CTA: 한 가지 행동으로 마무리

## 6. 카드와 리스트

- 혜택은 3개 또는 5개 단위가 안정적입니다.
- 카드 안에는 제목 1개, 설명 1개, 보조 근거 1개를 넘기지 않는 것이 좋습니다.
- 긴 기능 설명보다 “고객이 얻는 결과”를 먼저 씁니다.

## 7. 색상

- 배경, 본문, CTA, 강조색의 대비를 충분히 확보합니다.
- CTA 색상은 페이지 안에서 가장 눈에 띄어야 합니다.
- 고지/주의 문구는 너무 옅게 처리하지 않습니다.
- 색상만으로 상태를 구분하지 말고 텍스트도 함께 제공합니다.

## 8. 이미지

- 제품/인물/상황 이미지에는 `alt`를 제공합니다.
- 가격, 날짜, 환불 조건, 긴 설명은 이미지가 아니라 HTML 텍스트로 둡니다.
- 이미지 안 텍스트는 짧은 제목 수준으로 제한합니다.
- 모바일 로딩을 위해 이미지 용량을 점검합니다.

## 9. 실패 기준

아래 중 하나라도 있으면 100점이 아닙니다.

- 첫 화면에서 무엇을 제공하는지 3초 안에 알 수 없음
- CTA가 여러 개로 분산되어 사용자가 무엇을 해야 할지 모름
- 390px 폭에서 글자나 버튼이 깨짐
- 폼이 있으나 개인정보 동의 문구가 없음
- 가짜 후기처럼 보이는 문구가 있음
- “100% 보장”, “무조건”, “국내 1위” 등 근거 없는 표현이 있음
