# 한국형 모바일 LP 플레이북

이 문서는 한국 실무자가 바로 판매·교육·배포에 사용할 수 있는 모바일 LP 설계 기준입니다.

## 1. 기본 구조

한국형 모바일 LP는 보통 아래 흐름이 안정적입니다.

1. 히어로: 누구에게 무엇을 약속하는지
2. 문제 공감: 고객이 겪는 상황을 짚기
3. 해결책: 상품/서비스가 어떻게 돕는지
4. 혜택: 기능보다 결과 중심으로 설명
5. 신뢰 근거: 후기, 사례, 숫자, 경력, 포트폴리오
6. 구성/가격: 무엇을 받는지 명확히
7. FAQ: 망설임 해소
8. 최종 CTA: 한 가지 행동으로 유도

## 2. 첫 화면 체크

첫 화면에는 아래 4가지가 있어야 합니다.

- 대상: 누구를 위한 것인가?
- 약속: 어떤 결과를 기대할 수 있는가?
- 근거: 왜 믿을 수 있는가?
- 행동: 무엇을 누르면 되는가?

## 3. CTA 원칙

- CTA는 기본적으로 하나만 둡니다.
- 버튼 문구는 구체적으로 씁니다.
- “문의하기”보다 “무료 상담 신청하기”가 더 명확합니다.
- 하단 고정 CTA를 사용하면 모바일 전환에 유리합니다.
- 업종별 문구는 `reference/cta-patterns.md`를 참고합니다.

## 4. 한국어 카피 원칙

좋은 문장:

- “처음 시작하는 분도 2시간 안에 신청 페이지 초안을 완성합니다.”
- “복잡한 설정 없이 바로 따라 할 수 있는 실습형 강의입니다.”
- “상담 전 가격과 진행 방식을 먼저 확인할 수 있습니다.”

주의 문장:

- “무조건 매출이 오릅니다.”
- “100% 성공을 보장합니다.”
- “업계 1위입니다.” 단, 객관 근거가 없으면 사용 금지

## 5. 법적 주의

- 건강/의료: 치료, 완치, 질병 개선 표현 주의
- 화장품: 의약품처럼 보이는 효능 표현 주의
- 금융/투자: 수익 보장, 원금 보장 표현 금지
- 교육/취업: 합격 보장, 취업 보장 표현 주의
- 후기: 실제 후기와 출처가 있어야 안전함
- 개인정보: 신청 폼에는 처리 목적, 수집 항목, 보유 기간, 처리방침 링크를 안내

자세한 문구는 `reference/compliance-templates.md`를 따릅니다.

## 6. 디자인 원칙

- 모바일 폭 360~430px 기준
- 한 섹션에는 하나의 메시지만
- 버튼은 충분히 크게
- 배경과 텍스트 대비 확보
- 이미지 안 텍스트는 짧게
- 자주 바뀌는 정보는 HTML 텍스트로 처리

자세한 기준은 `reference/design-system.md`를 따릅니다.

## 7. 납품·교육용 산출물 기준

실무자가 클라이언트나 수강생에게 전달할 때는 아래가 함께 있어야 합니다.

- `lp-spec.md`: 승인된 사양서
- `index.html`: 실제 LP 파일
- `assets/`: 이미지와 정적 자산
- QA 결과표: 기능, 디자인, 모바일, 카피·법적 리스크, 배포 준비
- 확인 필요 목록: CTA 링크, 개인정보 처리방침 URL, 후기 근거, 가격/환불 조건
- 배포 안내: GitHub Pages 또는 Vercel 선택지

## 8. 완성 실패 기준

아래 중 하나라도 있으면 제품판이 아닙니다.

- 타깃이 모호함
- CTA가 여러 개로 분산됨
- 개인정보 수집 폼에 동의 문구가 없음
- 근거 없는 보장·성과·후기 문구가 있음
- 모바일 390px에서 깨짐
- 배포 전 승인 절차가 없음
